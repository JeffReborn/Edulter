## 一、产品背景

教培/教育咨询公司内部通常存在以下问题：

1. 内部资料分散，顾问很难快速找到统一、准确的标准答案。
2. 顾问与客户聊天后，客户信息主要停留在非结构化对话中，不利于后续跟进。
3. 跟进消息高度依赖顾问个人经验，表达不统一、效率低、质量波动大。
4. 新顾问上手慢，老顾问经验难沉淀为可复用的知识资产。

本 Demo 目标是验证：  
是否可以通过一个轻量级 AI 工具，帮助教育咨询顾问完成 **知识问答、客户画像提取、跟进消息生成** 三类高频工作。

---

## 二、产品目标

构建一个面向教育咨询/教培咨询公司的 AI 服务 Demo，验证以下核心价值：

- 让顾问可以基于内部资料快速获得标准答案与出处
- 让顾问可以从聊天记录中自动提取结构化客户画像
- 让顾问可以快速生成标准化、可编辑的跟进消息草稿

## 四、核心使用场景

### 场景 1：知识问答
顾问需要快速查询内部资料，例如：
- 某类项目/课程的标准介绍
- 某国家/地区的申请要求
- 服务流程与签约说明
- 常见问题的标准答复

系统应返回：
- 标准答案
- 相关出处
- 来源文档信息

### 场景 2：客户画像提取
顾问将客户聊天记录粘贴到系统中，系统自动识别并提取：
- 客户基本信息
- 学生阶段
- 目标国家/方向
- 预算
- 时间线
- 家长关注点
- 风险点
- 当前成交阶段

**客户标识与显示名（与实现对齐的产品约定）**

- **新建客户画像**与**在已有客户上更新画像**必须分流：更新须携带系统客户标识（`clientId`），**不得**仅靠显示名匹配同一客户。
- 客户 **显示名在系统内全库唯一**；顾问手动填写时须满足最短长度要求；若不填，系统生成唯一占位显示名。
- 顾问可在**客户列表**中修改显示名，便于将占位名改为更好记的名称。

### 场景 3：跟进消息生成
顾问基于客户画像与最近聊天内容，生成后续跟进话术，例如：
- 微信简短版
- 稍正式版
- 可选英文版

## 五、核心功能

### 1. 知识问答
输入：
- 上传后的内部资料
- 顾问提问

输出：
- 回答内容
- 出处引用
- 来源文档名

### 2. 客户画像提取
输入：
- 顾问与客户的聊天记录
- （新建场景）可选的客户显示名；**不填则系统生成唯一占位名**
- （更新场景）客户标识 `clientId`，用于在已有客户上合并更新画像

输出：
- 结构化客户画像
- 关键关注点
- 风险提示
- 当前跟进阶段判断

### 3. 跟进消息生成
输入：
- 客户画像
- 最近聊天记录
- 企业知识库中的标准表述

输出：
- 跟进消息草稿
- 可编辑文本
- 多种语气版本

## 六、输入与输出定义

### 输入方式
- 上传内部资料（PDF / DOCX / TXT，第一版可逐步支持）
- 粘贴客户聊天记录
- 在问答框中输入问题

### 输出方式
- 标准答案 + 出处
- 结构化客户画像
- 跟进消息草稿

---

## 七、成功标准（Demo 阶段）

### 产品成功标准
- 能跑通完整演示闭环：
  - 上传资料
  - 知识问答
  - 画像提取
  - 跟进消息生成

### 功能成功标准
- 知识问答能返回答案与至少一条出处
- 客户画像输出为固定结构，而非自由散文
- 跟进消息至少支持两种风格版本
- 主要页面具有基础 loading / error / empty state

### 演示成功标准
- 可向潜在客户或朋友公司展示完整流程
- 能使用一组 demo 数据进行稳定演示
- 输出结果整体具备业务可理解性与实用性

---

## 八、非功能要求

### 可用性
- 页面流程简单，顾问无需培训即可理解
- 结果可复制、可查看、可编辑

### 稳定性
- 主要 API 出错时有清晰提示
- 长文本输入时系统不应直接崩溃

### 可扩展性
- 架构允许未来增加：
  - 更多资料类型
  - 更多输出模板
  - 历史记录
  - 管理后台
  - 多租户能力
