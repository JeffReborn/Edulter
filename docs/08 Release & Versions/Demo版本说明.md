## 三、当前版本定位

当前版本定位为：

**一个面向教育咨询 / 教培咨询场景的企业 AI Demo，用于验证高频顾问工作流中的 AI 辅助价值。**

它的目标不是第一天就成为完整企业 SaaS，而是验证以下三个问题：

1. 企业内部资料是否能被用于稳定的知识问答
2. 咨询文本是否能被提取为有业务价值的客户画像
3. 客户画像是否能驱动自然、可发送的跟进消息生成

---

## 四、当前版本不是什么

当前版本 **不是**：

- 完整 CRM 系统
- 完整企业管理后台
- 多租户 SaaS 平台
- 自动化营销或自动触达系统
- 高级 BI 分析平台
- 全流程教培业务系统

这意味着当前版本不追求：

- 复杂权限
- 复杂组织管理
- 自动跟进任务
- 自动提醒
- 顾问分配
- 审批流
- 企业级配置后台

## 五、当前版本核心价值

当前 Demo 的核心价值集中在三条主线上：

### 1. 知识问答
顾问可以基于内部资料快速提问，并获得：
- 标准答案
- 出处引用
- 来源文档信息

### 2. 客户画像提取
顾问可以将咨询文本转成：
- 结构化客户画像
- 主要关注点
- 风险点
- 当前阶段判断

### 3. 跟进消息生成
顾问可以基于客户画像与咨询上下文生成：
- 微信简短版消息
- 稍正式版消息

## 七、当前版本的最小演示闭环

当前 Demo 的最小闭环定义如下：

1. 上传内部资料
2. 系统处理资料并入库
3. 基于资料进行知识问答
4. 粘贴咨询文本
5. 提取客户画像
6. 基于画像生成跟进消息
7. 把结果保存到客户记录中
8. 在客户记录中查看最新结果

只要这条链路稳定成立，当前 Demo 就已经具备核心演示价值。

## 八、当前版本应交付的能力

### A. 知识库侧能力
- 文档上传
- 文档处理状态展示
- 至少一种主要格式支持

### B. 客户侧能力
- 咨询文本输入
- 客户画像提取
- 客户主体创建 / 更新
- 多轮咨询下的手动追加更新能力（轻量版）
- 客户记录列表
- 客户详情查看

### C. 跟进行动能力
- 跟进消息生成
- 至少两种风格
- 结果复制
- 结果持久化保存

### D. 基础产品体验
- loading state
- error state
- empty state
- 基础页面可用性
- 基础结果可回看

## 十二、当前版本质量标准

当前 Demo 不追求“企业级完美”，但应满足以下最低质量标准：

### 1. 功能可演示
每个核心能力都不是假页面，而是真正跑通一条基础链路。

### 2. 输出可读、可用
- 问答答案能看懂
- 画像字段能用
- 跟进消息可直接微调后发送

### 3. 结构稳定
- 字段名稳定
- Schema 稳定
- 前后端不靠猜字段联调

### 4. 错误可感知
- 失败时有明确提示
- 空结果时有空状态
- 不出现无反馈“死掉”的页面

### 5. 范围可控
- 不因演示焦虑而无限加功能
- 不把 Demo 写成重型平台

## 十五、当前版本的对外沟通口径建议

如果需要向朋友、试点客户或合作方描述当前版本，可统一按以下口径理解：

### 一句话版
这是一个面向教育咨询顾问场景的 AI Demo，当前重点解决“查资料、整理客户信息、生成跟进消息”三件事。

### 展示版
当前版本不是完整 CRM 或企业平台，而是一个围绕顾问高频动作构建的 AI 工作台雏形，重点验证内部知识问答、客户画像提取和跟进消息生成的价值。

### 内部开发版
当前版本只要求主闭环真实跑通，不做大而全平台能力，优先保证结构清晰、范围稳定、可继续迭代。

## 十六、当前版本的风险提醒

### 1. 容易被误解为“只差一点就能商用”
当前版本仍然是 Demo，不应对外暗示已经具备完整商用品质。

### 2. 容易在演示前不断加功能
越接近演示，越容易临时加功能。  
当前版本应坚持“主闭环优先”。

### 3. 容易把客户记录模块做重
客户记录模块是承载结果的轻工作台，不是 CRM。

### 4. 容易低估 Prompt / Schema 稳定性的重要性
当前真正影响演示质量的，很大程度不是页面，而是输出结构和结果稳定性。

---

## 十七、当前版本开发提醒

在当前 Demo 阶段，所有新增开发需求都应先问：

1. 是否直接增强三条核心主线？
2. 是否有助于演示主闭环？
3. 是否会拖慢当前交付？
4. 是否更适合放入后续版本路线？

如果一个需求更像“平台化扩张”，则默认不进入当前版本。
