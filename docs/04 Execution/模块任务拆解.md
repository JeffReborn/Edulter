## 一、文档目标

本文件用于把当前项目中的核心模块，进一步拆成更适合实际开发和交给 Cursor 执行的任务卡。

它的作用是：

- 把模块说明转成开发动作
- 把大任务压缩成可执行的小闭环
- 为 Cursor 提供更明确的实施入口
- 降低“一次做太多导致跑偏”的风险
- 让你可以按模块逐步推进，而不是靠感觉开发

---

## 二、使用原则

### 1. 一张任务卡只做一个相对独立的闭环
不要把多个不同模块或多层职责混在同一张任务卡里。

### 2. 每张任务卡都要能回答 4 个问题
- 本次做什么
- 本次不做什么
- 验收标准是什么
- 依赖哪些文档

### 3. Cursor 一次只执行一张任务卡
不要把整份文档全部扔给 Cursor 后让它自己决定顺序。

### 4. 优先按当前开发顺序推进
当前建议顺序仍然是：

1. 项目骨架
2. 数据模型
3. 文档上传
4. 知识问答
5. 客户画像提取
6. 跟进消息生成
7. 客户记录

## 六、当前执行建议

### 1. 一次只给 Cursor 一张任务卡
不要把多张卡一起扔过去。

### 2. 任务卡完成后立刻验证
至少检查：
- typecheck
- lint
- 接口字段
- 基础运行

### 3. 每张任务卡完成后建议 commit
这样后面即使出现问题，也方便回退。

### 4. 如执行中出现边界冲突，及时更新文档
相关更新位置通常包括：
- [[阻塞问题]]
- [[产品决策]]
- [[架构决策]]
- [[模型与Prompt决策]]
