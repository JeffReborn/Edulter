## 一、文档目标

本文件用于管理本周实际执行的任务，是从 [[任务总表]] 和 [[当前Sprint]] 中下沉出来的本周行动清单。

它的作用是：

- 把“总任务池”压缩成本周真正要做的内容
- 明确每天优先级
- 避免本周目标过多、失焦
- 让 Cursor 协作有清晰任务入口
- 让本周结束时可以直接复盘“做了什么、还差什么”

---

## 二、本周核心目标

本周最核心的目标不是把整个 Demo 全做完，而是完成以下三件事：

### 目标 1：文档基线定稿到“可开工”状态
也就是：
- 产品边界清晰
- 技术边界清晰
- 模块边界清晰
- 任务拆分清晰

### 目标 2：项目代码骨架初始化完成
也就是：
- Next.js 项目跑起来
- Prisma 接通
- 数据库模型落下去
- 基础目录结构搭好

### 目标 3：文档上传模块最小闭环开工并尽量跑通
也就是：
- 上传文件
- 存文档记录
- 提取文本
- chunk 入库
- 状态展示

## 八、本周最大风险提醒

### 1. 任务铺太多
本周最容易犯的错误是：
- 一边写问答
- 一边写画像
- 一边写跟进
- 一边还做客户列表

结果每个都只开了头。

### 2. 文档陷入无限细化
当前文档只需要达到“足够支撑开发”的程度，不需要写成论文。

### 3. 文件解析成为阻塞点
文档上传链路里，文件解析很可能是第一个真实技术难点，要优先验证。

### 4. Cursor 一次改动范围过大
本周应坚持：
- 一次只给 Cursor 一个明确任务
- 每完成一小段就 commit
